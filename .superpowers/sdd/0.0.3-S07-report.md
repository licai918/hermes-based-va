## What this slice delivers

`hermes-runtime/hermes_runtime/knowledge/ingest.py` — Stage B of the two-stage
ingestion (FR-2): consumes an operator-pulled `corpus.json` → chunks (~600-char
word-boundary, identical algorithm to the spike's `probe/ingest.py`) → runs the
boundary check (NFR-3 / audit finding 3) → embeds the surviving chunks via an
injected `embed_fn` seam (default: fastembed `BAAI/bge-small-en-v1.5`,
`passage_embed`) → idempotently truncate-and-reloads `knowledge_chunk` in the
separate `toee_knowledge` DB. CLI: `python -m hermes_runtime.knowledge.ingest
<corpus.json>`, self-provisioning via `migrate()` (ensure_database + apply
migrations) first, mirroring `migrate.py`'s `main()` pattern.

Stage A (the Shopify pull itself) stays an operator step, documented in the
module docstring — the pull-artifact JSON shape
(`[{page_id, page_type, title, url, text}]`) is the contract; this module never
talks to Shopify.

### Boundary check (`check_boundaries`)

- **Policy-slot duplicate** (excluded from index): chunk text is a verbatim
  substring match (either direction) against a **published** Required
  Operational Policy Slot text, read from `workbench_policy_slot` in the
  business datastore (`toee_va`) — only when `TOOL_BACKEND=datastore`; returns
  `[]` gracefully on mock/unset backend, unreachable Postgres, or a
  not-yet-migrated table. Slots are currently all empty (verified live), so
  this branch never fires today — proven by a live test that temporarily
  writes a marker into `business-hours.published_text`, reads it back, then
  restores `NULL`.
- **Live-fact pattern** (flagged, still indexed): a `$N`/`CAD N` price-like
  regex or an "N in stock" / "N units available/left" stock-count regex.
- A `ponytail:` comment marks the substring-containment heuristic's ceiling
  (no normalization/fuzzy match) and its upgrade path, since it's fine only
  while slots are empty/short.

### Embedding

`fastembed_passage_embedder()` lazily imports `numpy`/`fastembed` (never
imported when a test supplies a fake `embed_fn`, and never built at all when
there's nothing to embed). Vectors serialize as **float32, native byte order**
(`np.asarray(v, dtype=np.float32).tobytes()`); documented inverse read is
`np.frombuffer(blob, dtype=np.float32)`. `embed_fn` is an injected seam so
production always uses the real model but tests use a deterministic fake.

### Idempotency / isolation

`ingest()` always issues `TRUNCATE knowledge_chunk RESTART IDENTITY` on
whatever connection it's given — that connection is always the knowledge DSN
(injected via `conn=`, or opened from `KNOWLEDGE_DATABASE_URL`/`dsn=` when
owned), so the TRUNCATE never has a path to the business database.

## Self-review

- **Boundary check heuristic** is intentionally simple (regex + substring
  containment), as directed. The containment check's false-positive ceiling
  is marked with a `ponytail:` comment; low risk today since all six policy
  slots are empty. Worth a second look once real policy text is authored and
  ingestion starts running against non-empty slots (S12 knowledge ADR is the
  named place for that governance question).
- **`_read_policy_slot_texts` reads directly from Postgres**, not through the
  `toee_knowledge_ops` tool-dispatch handler — deliberate: this is an offline
  batch job, not a turn, and going through the dispatch surface would need a
  fabricated `ToolExecutionContext`/actor for a read that doesn't need one.
  Documented in the function's docstring.
- **Did not wire a front-end change.** The task brief's "② E2E (browser)"
  layer (an `/admin/knowledge` entry showing corpus presence) is out of scope
  for this slice per the brief itself — S11 owns the status panel. The
  three-layer gate note in the brief already scopes it that way; flagging
  here so it isn't mistaken for an oversight.
- **No new dependency added.** `fastembed`/`numpy`/`psycopg` were already in
  the hermes-runtime venv (spike + S06); nothing new in `pyproject.toml`.
- Ran the full 380-test hermes-runtime suite once (89s) — green, no
  regressions from adding this module.
