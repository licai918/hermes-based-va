## The two reconnect shapes

**(1) Composio-managed connections (Shopify/QBO/Square) — OAuth reconnect.**
A governed admin action `toee_integrations.initiate_reconnect` generates the
connected-account re-auth link via the Composio SDK. The browser is redirected to the
provider, re-grants, and the provider returns to the workbench callback route, which
verifies a session-bound `state` and lands back on `/admin/integrations`; the page
then re-probes that row. The acting admin is recorded
(`integration_reconnect_initiated`). **No token touches the workbench — Composio holds
the credentials; we hand the admin a link and get one back.**

**(2) Static-token integrations (EasyRoutes/SimpleTexting/OpenRouter/Gadget) — guided
instructions + re-probe, NOT self-service token replacement** (gap-review P3, binding).
There is no OAuth and the workbench cannot edit deployment env vars. The panel names the
env var (already in the row `detail`) and where it lives; the operator rotates it in the
deployment env, then clicks "Re-probe now" (`toee_integrations.reprobe_now`, which
reuses S16's single-integration probe and records + audits the result,
`integration_reprobed`). **No token value is ever displayed, entered, or stored outside
env.** `reprobe_now` serves both shapes' completion so a reconnected badge refreshes
immediately rather than on the next 15-min cycle.

Files: handlers `hermes-runtime/hermes_runtime/datastore/handlers/integrations.py`
(`_initiate_reconnect`, `_reprobe_now`); SDK wire
`hermes/toee_hermes/drivers/composio/driver.py` (`initiate_composio_reconnect` +
`_reauth_redirect_url`); probe seam

## The callback route + its state-binding security

`GET /api/admin/integrations/callback` is a **new, externally-reachable route**, so its
input is treated as untrusted. Security rests on one bound `state`
(`apps/workbench/lib/bff/admin/reconnect-state.ts`):

- **State binding.** `initiate_reconnect`'s route mints a random `state`, sets it as an
  **httpOnly, sameSite=lax** cookie scoped to `/api/admin/integrations`, and appends it
  to the `callback_url` it builds from its OWN origin (never client-supplied). The
  provider round-trips the state.
- **Verify + redirect, nothing else.** The callback compares the query `state` to the
  cookie and does nothing but verify + redirect — no token handling, and it acts on no
  other callback parameter. `sameSite=lax` (not strict) so the cookie survives the
  top-level GET redirect back from the provider; `httpOnly` so page JS cannot read/forge
  it; the one-time cookie is always cleared on return.
- **State-mismatch refusal — confirmed.** A missing cookie, a missing query value, or
  any mismatch is **refused**: the page is redirected with `?reconnect=state_mismatch`
  and never signals success (the panel shows a fail-closed error). Pinned by
  `reconnect-state.test.ts` (the `reconnectStateMatches` fail-closed cases).
- Admin-only: the callback sits under `/api/admin/integrations`, so an invalid/absent
  admin session is 401/403'd (middleware + `withSession`) before the state check runs.

A reconnect that cannot complete (Composio SDK error, owner-blocked creds, state
mismatch) surfaces a governed error, never a false "reconnected".

## How secrets are kept out (NFR-6)

No secret is displayed, entered, logged, or committed. The static-token flow shows env-var
NAMES + instructions only (from the existing `detail`), never a value. The Composio flow
handles no token — Composio holds the credentials; `initiate_reconnect` returns only a
redirect URL, and the callback handles no token. Probe results carry status + a
secret-free reason only. `node scripts/secret-scan.mjs --selfcheck && node scripts/secret-scan.mjs`
→ clean (31 selfcheck cases).

The Composio re-auth link generation (`initiate_composio_reconnect` →

`composio_api_error`, no fabricated link, no audit row). It must be confirmed against a
live connected account at cutover — exactly like the `_ComposioSdkClient` execute path

verification, audit attribution, role gating, and the static-token re-probe path — is
fully built and tested independent of the live SDK. No live reconnect is fabricated.
